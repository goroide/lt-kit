LTKit

A SwiftUI component library and design system for the Life Tracker app series. Shared UI, storage, and configuration in one Swift Package.

Swift 5.9+ iOS 16+ macOS 13+ SwiftUI

Installation

Add LTKit as a local Swift Package dependency via Xcodegen or Xcode's Package Manager.

project.yml (Xcodegen)

packages:
  LTKit:
    path: ../LTKit

targets:
  MyApp:
    dependencies:
      - package: LTKit
        product: LTKit

Package.swift

.package(url: "https://github.com/goroide/lt-kit", from: "1.0.0")

LTSettingsView

Pre-built settings screen: History link, Privacy Policy, and Clear Data button. Fully configured via AppConfig.

LTSettingsView

Settings screen with history, privacy, and data controls

History ›
Privacy Policy ›
Clear Data
import LTKit

LTSettingsView(
    config: AppConfig.current,
    onClearAllData: { viewModel.clearAllData() }
) {
    HistoryView(viewModel: viewModel)
}

If AppConfig.privacyPolicyURL is set, Privacy Policy opens that URL. Otherwise it shows the built-in LTPrivacyPolicyView.

LTAppFlow

Manages the standard app startup sequence: Splash → FirstTime (if data is empty) → Home. Handles the splash timer and @AppStorage flag automatically.

LTAppFlow

Splash → FirstTime / Home routing

import LTKit

LTAppFlow(
    splashDisplayName: AppConfig.current.appDisplayName,
    splashLogoName: "SplashLogo",
    splashSubtitle: "Life Tracker Lab",
    hasSeenSplashKey: "hasSeenSplash",
    isEmpty: viewModel.habits.isEmpty
) {
    HomeView(viewModel: viewModel)           // shown when data exists
} firstTime: {
    FirstTimeView(viewModel: viewModel)      // shown on first launch
}

Parameters

ParameterTypeDefaultDescription
splashDisplayNameString—App name shown on splash
splashLogoNameString"SplashLogo"Logo image asset
splashSubtitleString""Subtitle on splash
hasSeenSplashKeyString—UserDefaults key for splash flag
isEmptyBool—Whether to route to firstTime
home@ViewBuilder—Content shown when data exists
firstTime@ViewBuilder—Content shown on first launch

⚠️ Note: If your app clears all data and needs to re-show the FirstTime screen dynamically, use LTSplashView directly rather than LTAppFlow, as onAppear does not re-trigger after initial render.

Colors

Semantic color tokens that adapt automatically to Light / Dark mode via UIKit system colors. Use Color.ltXxx in SwiftUI.

Token Light Dark Usage
Color.ltSurface #FFFFFF #000000 Screen / sheet background
Color.ltOnSurface #000000 #FFFFFF Primary text and icons
Color.ltOnSurfaceSecondary #6E6E73 #8E8E93 Secondary text, captions
Color.ltSeparator #C6C6C8 #38383A Dividers and borders

Hex initializer

// Color(hex:) is provided by LTKit
Color(hex: "#FF81CC")
Color(hex: "FF81CC")   // # prefix optional

JSONStore

Generic, Codable-backed JSON file storage. Reads and writes to the app's Documents directory by default.

JSONStore<T: Codable>

Type-safe persistent JSON storage

import LTKit

// In a ViewModel
private let habitStore = JSONStore<[Habit]>(fileName: "habits.json")
private let logStore   = JSONStore<[HabitLog]>(fileName: "habitLogs.json")

// Load
habits = habitStore.load() ?? []

// Save
habitStore.save(habits)

// Clear
habitStore.clear()

// For testing — inject a temp directory
let store = JSONStore<[Habit]>(fileName: "habits.json", directory: tempDir)

API

Method / InitDescription
init(fileName:directory:)Creates store. directory defaults to Documents.
load() -> T?Decodes and returns stored value, or nil if missing/corrupt.
save(_ value: T)Encodes and writes value to disk.
clear()Deletes the underlying file.

AppConfig

Per-app configuration struct. Define one static let current per app target using an extension.

AppConfig

App identity, colors, storage keys, and URLs

// AppConfig+App.swift (in your app target)
import SwiftUI
import LTKit

extension AppConfig {
    static let current = AppConfig(
        appName: "CoreHabit",
        appDisplayName: "CORE HABIT",
        bundleId: "com.goroide.corehabit",
        accentColor: Color(hex: "#FF81CC"),
        storageKeys: .init(
            primaryFile: "habits.json",
            secondaryFile: "habitLogs.json"
        ),
        privacyPolicyURL: nil,    // nil = use LTPrivacyPolicyView
        appStoreURL: nil
    )
}

Properties

PropertyTypeDescription
appNameStringInternal app identifier
appDisplayNameStringUser-visible name (splash, settings)
bundleIdStringBundle identifier
accentColorColorApp accent color
storageKeys.primaryFileStringPrimary JSON file name
storageKeys.secondaryFileString?Optional secondary JSON file
privacyPolicyURLURL?External privacy policy URL (nil = built-in view)
appStoreURLURL?App Store link (optional)
